# 產品需求文件 (PRD)：活動報名系統

## 1. 專案概述

### 背景與動機
大學校園內常有各式各樣的活動，但傳統的報名方式往往依賴紙本、表單或是沒有統一管理機制的平台，導致主辦方難以管理報名人數與候補狀況，學生也難以即時掌握自己的報名結果或候補進度。為了解決這些痛點，本專案旨在開發一個自動化報名管理系統，讓學生報名流程更加順暢，並由系統自動處理候補名單，減輕主辦方的人工作業負擔。

### 目標用戶
- **學生（報名者）：** 瀏覽校園活動、進行報名、查詢結果與關注候補狀態的學生。
- **活動主辦方（管理員）：** 建立並管理活動資訊、查看報名與候補名單的使用者（例如系學會、社團幹部）。

### 核心價值主張
簡化報名流程，提供透明且自動化的候補機制，提升學生的報名體驗，同時降低主辦方的管理成本。

---

## 2. 功能需求

以下為系統預計提供的主要功能與對應的使用者故事：

### 1. 建立活動
- **描述：** 允許主辦方建立新的活動頁面，並設定活動詳細資訊與容納人數。
- **使用者故事：** 作為主辦方，我希望能夠在系統建立活動（包含名稱、時間、地點、人數上限與簡介），以便學生可以了解活動並進行報名。

### 2. 顯示報名人數
- **描述：** 在活動頁面即時顯示目前已報名的人數及剩餘名額。
- **使用者故事：** 作為學生，我希望能在活動頁面看到目前的報名進度與剩餘名額，以便決定是否要立即報名。

### 3. 活動資訊瀏覽
- **描述：** 提供活動列表與詳細內頁，讓學生瀏覽各項活動資訊。
- **使用者故事：** 作為學生，我希望能夠瀏覽所有開放報名的活動清單與詳細資訊，以便尋找我有興趣參加的活動。

### 4. 自動候補機制
- **描述：** 當活動報名人數滿額後，後續的報名者將自動排入候補名單；若有正取者取消報名，系統依序自動將候補遞補為正取。
- **使用者故事：** 作為主辦方，我希望系統能在名額滿時自動建立候補名單，並在有人取消時自動遞補，以便減少人工聯絡的成本。
- **使用者故事：** 作為學生，我希望在名額滿時還可以登記候補，並在有名額釋出時自動獲得參加資格，以便不會錯過喜歡的活動。

### 5. 報名結果查詢
- **描述：** 學生可以查詢個人所有報名的活動狀態（如：正取、候補中、已取消）。
- **使用者故事：** 作為學生，我希望可以有一個專屬頁面查詢我自己報名過的所有活動及目前的狀態，以便隨時掌握參與進度。

## 3. 非功能需求

### 技術限制
- **系統架構：** 基於 Python 的 Flask 框架開發。
- **前端技術：** 使用 HTML 搭配 Jinja2 模板引擎與原生 CSS / JavaScript 進行渲染。
- **資料庫：** 使用 SQLite 進行資料儲存與管理。

### 效能與安全考量
- **效能：** 頁面載入時間需控制在合理範圍內，確保高併發時（例如熱門活動搶票）資料庫的鎖定機制能正確處理報名順序，避免超收。
- **安全：**
  - 使用者需登入才能報名與建立活動（實作基本的身分驗證機制）。
  - 對表單輸入進行防護，防止 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

| 功能分類 | 優先層級 | 功能項目 |
| :--- | :--- | :--- |
| **Must Have**<br>(核心必備功能) | 高 | 1. 活動資訊瀏覽 (活動列表與內頁)<br>2. 建立活動 (主辦方新增活動)<br>3. 基礎報名功能 (報名、取消報名)<br>4. 顯示報名人數 (已報名人數與上限)<br>5. 報名結果查詢 |

| **Nice to Have**<br>(未來擴充或優化功能) | 低 | 1. 活動分類與搜尋功能<br>2. 報名成功/遞補成功 Email 通知系統<br>3. 報名名單匯出 (Excel / CSV) 功能 |
